Normalization of Bookings Table

Question:

Given the unnormalized Bookings table, perform normalization to 1NF, 2NF, 3NF, and BCNF.

Answer by Rathna Sai Teja Panguluri (811358987):

Unnormalized Table:

The Bookings table includes multiple seat numbers for the same booking, which leads to redundancy and potential anomalies. The data structure is as follows:

BookingID CustomerID CustomerName FlightID FlightName Date Price SeatNumbers AirlineID AirlineName
B001 C001 Alice Brown F001 Flight A 2024-10-12 300 12A, 12B A001 SkyHigh
B002 C002 Bob White F002 Flight B 2024-10-13 250 15C A002 AirWings
B003 C001 Alice Brown F003 Flight C 2024-11-01 400 8A, 8B, 8C A003 FlyAway

1NF (First Normal Form):

To achieve 1NF, we eliminate repeating groups (i.e., multiple seat numbers in one column). Each seat number will have its own row.

BookingID CustomerID CustomerName FlightID FlightName Date Price SeatNumber AirlineID AirlineName
B001 C001 Alice Brown F001 Flight A 2024-10-12 300 12A A001 SkyHigh
B001 C001 Alice Brown F001 Flight A 2024-10-12 300 12B A001 SkyHigh
B002 C002 Bob White F002 Flight B 2024-10-13 250 15C A002 AirWings
B003 C001 Alice Brown F003 Flight C 2024-11-01 400 8A A003 FlyAway
B003 C001 Alice Brown F003 Flight C 2024-11-01 400 8B A003 FlyAway
B003 C001 Alice Brown F003 Flight C 2024-11-01 400 8C A003 FlyAway

2NF (Second Normal Form):

To achieve 2NF, we remove partial dependencies. The table is split into two: Bookings and Flights. The CustomerID and FlightID form a composite key in Bookings.

Bookings Table:

BookingID CustomerID FlightID SeatNumber
B001 C001 F001 12A
B001 C001 F001 12B
B002 C002 F002 15C
B003 C001 F003 8A
B003 C001 F003 8B
B003 C001 F003 8C

Flights Table:

FlightID FlightName Date Price AirlineID AirlineName
F001 Flight A 2024-10-12 300 A001 SkyHigh
F002 Flight B 2024-10-13 250 A002 AirWings
F003 Flight C 2024-11-01 400 A003 FlyAway

3NF (Third Normal Form):

To achieve 3NF, we remove transitive dependencies. The AirlineName is dependent on AirlineID, so we create a new Airlines table.

Airlines Table:

AirlineID AirlineName
A001 SkyHigh
A002 AirWings
A003 FlyAway

Updated Flights Table (after removing AirlineName):

FlightID FlightName Date Price AirlineID
F001 Flight A 2024-10-12 300 A001
F002 Flight B 2024-10-13 250 A002
F003 Flight C 2024-11-01 400 A003

BCNF (Boyce-Codd Normal Form):

In the current structure, all functional dependencies are preserved, and every determinant is a candidate key. Therefore, the table already satisfies BCNF.

Conclusion:

The final normalized database structure consists of three tables: Bookings, Flights, and Airlines, all in BCNF.
