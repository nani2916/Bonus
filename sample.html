3NF (Third Normal Form)

To achieve 3NF, we ensure that no transitive dependencies exist, meaning non-primary key columns should not depend on other non-primary key columns. The current design already follows this principle, as each table is dependent only on its primary key.

BCNF (Boyce-Codd Normal Form)

BCNF is achieved by further refining the 3NF structure to ensure that every determinant is a candidate key. In the current structure, all functional dependencies are satisfied, meaning that there are no overlapping candidate keys.
